# คู่มือการใช้งาน Animal Quiz Builder

คู่มือนี้อธิบายวิธีใช้งานระบบสร้าง Quiz แบบ personality profile ที่มีหลังบ้านเป็น graph editor สำหรับลากเส้นคำตอบไปยังคำถามหรือผลลัพธ์

## ภาพรวมระบบ

ระบบมี 2 โหมดหลัก

- `เล่น Quiz`: ใช้ทดสอบ quiz เหมือนผู้เล่นจริง ตอบคำถาม ดูผลลัพธ์ และแชร์ภาพ Story
- `หลังบ้าน`: ใช้สร้างและแก้ไข quiz ผ่าน canvas, graph route, score dimensions และ result profiles

## ส่วนประกอบในหน้าหลังบ้าน

### 1. Canvas กลาง

พื้นที่ตรงกลางคือ graph editor

- `Question node` คือกล่องคำถาม มีคำตอบอยู่ด้านใน
- `Result node` คือกล่องผลลัพธ์ เช่น โลมา จิ้งจอก สิงโต
- เส้น route คือเส้นที่บอกว่าถ้าตอบข้อนี้แล้วจะไป node ไหนต่อ
- ลาก node เพื่อจัดตำแหน่งบน canvas ได้
- ลบเส้นได้โดยเลือกเส้นแล้วกด `Delete` หรือ `Backspace`

### 2. Toolbar ด้านบนของหลังบ้าน

- `+ คำถาม`: เพิ่ม Question node ใหม่
- `+ ผลลัพธ์`: เพิ่ม Result node ใหม่
- ไอคอนบันทึก: บันทึก config ลงเครื่องทันที
- ไอคอนดาวน์โหลด: Export quiz เป็นไฟล์ JSON
- ไอคอนอัปโหลด: Import quiz จากไฟล์ JSON
- ไอคอนรีเซ็ต: รีเซ็ตกลับเป็น quiz ตัวอย่าง

### 3. Inspector ด้านขวา

Inspector คือแผงแก้ไขรายละเอียด

- คลิกพื้นที่ว่างบน canvas: แก้ `Quiz settings`
- คลิก Question node: แก้คำถาม คำตอบ route และคะแนนของคำตอบ
- คลิก Result node: แก้ชื่อผลลัพธ์ คำอธิบาย emoji สี และ result profile

## วิธีสร้าง Quiz ใหม่

### ขั้นที่ 1: ตั้งค่า Quiz settings

คลิกพื้นที่ว่างบน canvas เพื่อเปิด `Quiz settings`

ตั้งค่าหลัก

- `ชื่อ Quiz`: ชื่อแบบทดสอบ
- `description`: คำอธิบายสั้นๆ
- `คำถามเริ่มต้น`: เลือกว่า quiz จะเริ่มจากคำถามไหน
- `Score dimensions`: มิติคะแนน เช่น สังคม พลังงาน เมตตา อิสระ เหตุผล

ค่า `min` และ `max` ของแต่ละมิติใช้กำหนดช่วงคะแนน เช่น 0 ถึง 3

### ขั้นที่ 2: สร้าง Result nodes

กด `+ ผลลัพธ์` แล้วคลิก result node ที่สร้างขึ้นเพื่อแก้ไข

ควรตั้งค่า

- `ชื่อผลลัพธ์`: เช่น โลมา สิงโต นกฮูก
- `subtitle`: คำอธิบายสั้น
- `description`: รายละเอียดผลลัพธ์
- `emoji`: ภาพแทนผลลัพธ์
- `color`: สีประจำผลลัพธ์
- `Result profile`: คะแนนเป้าหมายของผลลัพธ์นั้นในแต่ละมิติ

ตัวอย่าง: ถ้า “โลมา” เป็นผลลัพธ์ที่เข้าสังคมสูงและเมตตาสูง ให้ตั้งมิติ `สังคม` และ `เมตตา` สูงกว่ามิติอื่น

### ขั้นที่ 3: สร้าง Question nodes

กด `+ คำถาม` แล้วคลิก question node เพื่อแก้ไข

ใน Question inspector แก้ได้

- ข้อความคำถาม
- note เพิ่มเติม
- รายการคำตอบ
- คะแนนของแต่ละคำตอบ
- route ของแต่ละคำตอบ

### ขั้นที่ 4: ตั้งคะแนนให้คำตอบ

ในแต่ละคำตอบ กด `เพิ่มคะแนน`

แต่ละ score effect มี 3 ส่วน

- มิติคะแนน เช่น สังคม เมตตา เหตุผล
- operation เช่น `average`, `add`, `set`
- value เช่น 0, 1, 2, 3

ความหมายของ operation

- `average`: เอาคะแนนเดิมกับค่าคำตอบมาเฉลี่ย เหมาะกับ personality quiz
- `add`: บวกคะแนนเพิ่ม เหมาะกับระบบสะสมแต้ม
- `set`: ตั้งค่ามิตินั้นเป็นค่าที่กำหนดทันที

## วิธีคิดผลลัพธ์

ระบบคำนวณคะแนนผู้เล่นจากคำตอบทั้งหมด แล้วนำไปเทียบกับ `Result profile`

สูตรที่ใช้คือ

```text
Σ |คะแนนผู้เล่น - โปรไฟล์ผลลัพธ์|
```

ผลลัพธ์ที่มีค่าระยะห่างน้อยที่สุดคือผลลัพธ์ที่ใกล้ผู้เล่นที่สุด

ถ้าลาก route จากคำตอบไปยัง Result node โดยตรง ระบบจะใช้ผลลัพธ์นั้นเป็นผลลัพธ์ปลายทาง แต่ยังคำนวณ ranking ให้ดูประกอบในหน้าผลลัพธ์

## วิธีลากเส้น route

1. ดูคำตอบใน Question node
2. ที่ด้านขวาของแต่ละคำตอบจะมีจุดเชื่อม
3. ลากจากจุดนั้นไปด้านซ้ายของ Question node หรือ Result node ที่ต้องการ
4. ถ้าคำตอบเดิมมี route อยู่แล้ว การลากใหม่จะเปลี่ยน route ของคำตอบนั้น

ตัวอย่าง flow

```text
Question 1 / Answer A -> Question 2
Question 1 / Answer B -> Question 3
Question 12 / Answer A -> Result โลมา
```

ถ้าคำตอบไม่มี route ระบบจะไปจบ quiz และเลือกผลลัพธ์จากคะแนนอัตโนมัติ

## วิธีทดสอบ Quiz

1. กด `เล่น Quiz` จากหน้าหลังบ้าน
2. ตอบคำถามตาม flow ที่ตั้งไว้
3. ดูหน้าผลลัพธ์
4. ตรวจส่วน `โปรไฟล์คะแนน`, `วิธีคิด`, และ `คำตอบและคะแนน`
5. ถ้าผลลัพธ์ไม่ตรง ให้กลับไปหลังบ้านแล้วปรับคะแนนหรือ result profile

ปุ่ม `เล่น Quiz` ในหลังบ้านคือโหมด preview ของ draft ในเครื่อง URL จะมี `?draft=1` ต่อท้ายเพื่อแยกจากหน้าเล่นจริง

## การแชร์ผลลัพธ์

หน้าผลลัพธ์มีปุ่ม

- `IG Story`
- `Facebook Story`
- `ดาวน์โหลด`

ระบบจะสร้างภาพ Story ขนาด 1080x1920 จากผลลัพธ์ปัจจุบัน

- ถ้าเปิดบนมือถือและ browser รองรับ Web Share API จะเปิด native share sheet ให้เลือกแอป
- ถ้าไม่รองรับ ระบบจะดาวน์โหลดภาพ Story เพื่อให้ผู้ใช้นำไปลงเอง

## การบันทึกและสำรองข้อมูล

ระบบบันทึกข้อมูลหลังบ้านลง `localStorage` อัตโนมัติใน browser ที่ใช้งาน ข้อมูลนี้เป็น draft เฉพาะเครื่องนั้น

เพื่อความปลอดภัยควร Export JSON เป็นระยะ โดยเฉพาะก่อนแก้ไขเยอะๆ

- ใช้ `Export JSON` เพื่อเก็บ config
- ใช้ `Import JSON` เพื่อโหลด config กลับมา
- ใช้ `รีเซ็ต` เฉพาะเมื่อแน่ใจ เพราะจะกลับไปเป็นตัวอย่างเริ่มต้น

ถ้าแก้คำถามหรือคำตอบในหลังบ้านแล้วเปิดบนมือถือไม่เห็นทันที แปลว่ายังเป็น draft ในเครื่องเดิม ต้อง publish เวอร์ชันนั้นก่อน มือถือที่เปิด URL หลักจะอ่านเฉพาะเวอร์ชันที่ deploy แล้ว ไม่อ่าน draft จากเครื่องอื่น

## การใช้ Firebase

ระบบเชื่อม Firebase project `animal-quiz-builder-mt` แล้ว โดยเก็บ quiz กลางไว้ที่ Firestore document นี้

```text
quizzes/main
```

หน้าเล่น Quiz จะอ่านข้อมูลจาก Firebase อัตโนมัติ ถ้า Firebase โหลดไม่ได้จะ fallback ไปใช้ quiz ตัวอย่างที่ build มากับเว็บ

ในหลังบ้านมีปุ่ม Cloud เพิ่มเติม

- `โหลดจาก Firebase`: ดึง quiz เวอร์ชันกลางมาเป็น draft ในเครื่อง
- `เผยแพร่`: ส่ง draft ปัจจุบันขึ้น Firebase ให้มือถือและผู้เล่นเห็น
- `Login/Logout`: เข้าสู่ระบบ Google/Firebase admin ก่อนเผยแพร่

Firestore rules ตั้งไว้แบบปลอดภัยคือทุกคนอ่านได้ แต่เขียนได้เฉพาะผู้ใช้ที่ login ด้วยอีเมล `multithai.info@gmail.com`

```text
read: ทุกคน
write: multithai.info@gmail.com เท่านั้น
```

ถ้ากดเผยแพร่แล้ว login ไม่ได้ ให้เปิด Firebase Console > Authentication แล้วเปิด provider แบบ Google และเพิ่ม authorized domain `multithai.github.io` ก่อน

## การเผยแพร่บน GitHub Pages

โปรเจกต์นี้ตั้ง GitHub Actions ไว้แล้ว

เมื่อ push code หรือ quiz config เวอร์ชัน publish ไปที่ branch `main`

1. GitHub Actions จะติดตั้ง dependencies
2. build แอปด้วย Vite
3. upload `dist`
4. deploy ไปที่ GitHub Pages

URL ปัจจุบันคือ

```text
https://multithai.github.io/animal-quiz-builder/
```

ลิงก์แยกตามโหมด

```text
หน้าเล่น Quiz: https://multithai.github.io/animal-quiz-builder/
หลังบ้าน Admin: https://multithai.github.io/animal-quiz-builder/admin
Preview draft ในเครื่อง: https://multithai.github.io/animal-quiz-builder/?draft=1
```

## Checklist ก่อนเผยแพร่

- คำถามเริ่มต้นถูกต้อง
- ทุกคำถามมีคำตอบครบ
- คำตอบสำคัญมีคะแนนครบ
- route หลักเชื่อมไปคำถามถัดไปหรือผลลัพธ์ถูกต้อง
- result profile ตั้งค่าครบทุกผลลัพธ์
- เล่นทดสอบอย่างน้อย 3-5 รอบด้วยคำตอบคนละแนว
- ทดลองปุ่มแชร์/ดาวน์โหลดภาพ Story
- Export JSON เก็บสำรองก่อน deploy
